# TypeScript `any`

> **Subject:** 「TypeScriptで`any`を使うべきではない。」

## FCoTなし

```
> TypeScriptで`any`を使うべきではない。

この方針の精神には概ね同意しますが、「絶対に使うな」は言い過ぎです。もう少し踏み込んで考えてみます。

`any`が危険な理由: TypeScriptの型システムを完全にオプトアウトしてしまいます。一度`any`がコードに入り込むと、それは伝染します——触れるすべての値が型チェックなしになってしまう。これはTypeScriptを使う目的そのものを無意味にします。

`unknown`はほぼ常に優れた選択肢です: 型が本当に分からない場合、`unknown`を使えば使う前に必ず型を絞り込まなければなりません。「まだ型が分からない」を型安全に表現できる方法です。

ただし、`any`が正当化されるユースケースも存在します:
- JavaScriptからの移行: すべてを一度に型付けするのは現実的ではありません。
- 型定義のないサードパーティライブラリ。
- TypeScriptの型システムでは表現できない複雑な型操作。
- 型付けがスピードを落とすような速度重視のプロトタイピング。

本当のルール: デフォルトでは`any`を避ける。使う場合は意図的な選択であるべき——怠慢の結果であってはならない。後で見直すべき技術的負債として扱うこと。ESLintの`@typescript-eslint/no-explicit-any`ルールで強制し、`any`が本当に正当化される箇所にのみ`eslint-disable`コメントを使うのがよいでしょう。
```
